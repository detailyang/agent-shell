# Design: Mouse Input

Generated by /think on 2026-05-15
Status: Draft

## Problem

Agent 需要与 TUI 程序（vim、htop、tmux 内嵌面板等）通过鼠标事件交互。当前 agent-shell 只支持文本输入和控制字符，无法发送点击、滚轮滚动或拖拽事件。这意味着 agent 无法完成"点击 vim 中某个位置"、"滚动 htop 列表"、"拖拽 tmux 面板分割线"等操作。

本设计新增 `mouse` 子命令，将鼠标事件封装为 SGR 编码序列写入 PTY，agent 只需指定坐标和动作类型。

## Out of Scope

- 查询 PTY 程序是否启用了鼠标模式（程序未启用时序列被忽略，这是可接受行为）
- 鼠标悬停（hover）事件——仅极少数程序响应纯移动，且 agent 无此需求
- 像素级坐标——仅支持字符单元坐标 (col, row)
- 就绪检测——鼠标事件是 fire-and-forget，不触发 send 的等待机制

## Constraints

1. 使用 SGR 编码（`CSI < Pb ; Px ; Py M/m`）作为唯一鼠标协议。理由：支持坐标 > 223，现代终端程序普遍支持。
2. 新增独立子命令 `mouse`，不修改现有 `send` 命令接口。
3. 坐标为 1-based (col, row)，与终端标准一致。
4. 必须支持：左/中/右键点击、滚轮上/下、press/release 分离、move（拖拽中）、drag 便捷语法。
5. 写入路径复用 `Session::pty_writer`，不引入新的 I/O 通道。

## Assumptions

1. 目标 TUI 程序已通过 `CSI ?1006h`（SGR 模式）或 `CSI ?1000h`/`CSI ?1002h`/`CSI ?1003h`（基础模式但兼容 SGR 输入）启用鼠标报告。若未启用，序列被忽略——这不是 bug。
2. Agent 知道目标程序的 PTY 尺寸（可通过 `list` 查询 rows/cols），因此能提供有效坐标。
3. 拖拽操作中，agent 负责提供合理的中间点路径（drag 命令会自动在起止点间插值）。

## Approach

### SGR Mouse Encoding

SGR 格式：`\x1b[<{button};{col};{row}{suffix}`

- `suffix`：`M` = press/move，`m` = release
- `button` 编码：
  - 0 = left press
  - 1 = middle press
  - 2 = right press
  - 32 = left drag (motion with button held)
  - 33 = middle drag
  - 34 = right drag
  - 35 = motion (no button, for `CSI ?1003h` all-motion mode)
  - 64 = scroll up
  - 65 = scroll down

### CLI Interface

```
agent-shell mouse --session <ID> <ACTION> [OPTIONS]

Actions:
  click    --x <col> --y <row> [--button left|middle|right] [--count <n>]
  scroll   --x <col> --y <row> --direction up|down [--count <n>]
  press    --x <col> --y <row> [--button left|middle|right]
  release  --x <col> --y <row> [--button left|middle|right]
  move     --x <col> --y <row> [--button left|middle|right]
  drag     --from-x <col> --from-y <row> --to-x <col> --to-y <row> [--button left|middle|right] [--steps <n>]
```

- `click` = press + release（便捷语法）
- `scroll` = 发送 `count` 次滚轮事件（默认 count=3）
- `press` / `release` / `move` = 原子事件，用于精确控制
- `drag` = press(from) → N 次 move(interpolated) → release(to)

### Protocol Addition

```rust
#[serde(rename = "mouse")]
Mouse {
    session_id: String,
    action: String,        // "click" | "scroll" | "press" | "release" | "move" | "drag"
    x: u16,               // col, 1-based
    y: u16,               // row, 1-based
    button: Option<String>,     // "left" | "middle" | "right", default "left"
    direction: Option<String>,  // "up" | "down" (scroll only)
    count: Option<u16>,         // repeat count (click/scroll)
    to_x: Option<u16>,         // drag target col
    to_y: Option<u16>,         // drag target row
    steps: Option<u16>,        // drag interpolation steps, default 5
}
```

### Data Flow

```
CLI (mouse --session X click --x 10 --y 5)
  │
  ▼ JSON over Unix socket
Daemon: handle_mouse(req)
  │
  ├─ validate session exists, not exited, not destroying
  ├─ validate coordinates (1 <= x <= cols, 1 <= y <= rows)
  ├─ encode action → Vec<Vec<u8>> (one or more SGR sequences)
  ├─ write all sequences to session.pty_writer
  ├─ record to recording if active
  └─ return Response::ok()
  │
  ▼
PTY (program receives SGR mouse events)
```

### Encoding Module

新增 `crates/core/src/mouse.rs`：纯函数模块，无状态。

```
pub fn encode_click(button, x, y, count) -> Vec<Vec<u8>>
pub fn encode_scroll(direction, x, y, count) -> Vec<Vec<u8>>
pub fn encode_press(button, x, y) -> Vec<u8>
pub fn encode_release(button, x, y) -> Vec<u8>
pub fn encode_move(button, x, y) -> Vec<u8>
pub fn encode_drag(button, from_x, from_y, to_x, to_y, steps) -> Vec<Vec<u8>>
```

每个函数返回要写入 PTY 的原始字节。`encode_drag` 在 from→to 之间做线性插值生成中间 move 事件。

### Session Method

`Session` 新增：

```rust
pub fn send_raw_bytes(&mut self, data: &[u8]) -> Result<(), String>
```

与 `send_text` 类似但不追加 `\n`。鼠标编码模块生成的字节通过此方法写入。此方法也可供未来其他原始输入需求复用。

## Key Decisions

1. **选择 SGR 编码而非 X10/UTF-8/urxvt 编码**：SGR 无坐标上限，press/release 可区分，是 2015 年后终端程序的事实标准。此决策在需要兼容极老程序（如 1990s curses 应用）时需要重新评估。

2. **独立子命令 `mouse` 而非扩展 `send`**：鼠标事件不触发就绪检测（无 nowait/timeout 语义），混入 `send` 会污染其接口。此决策在发现"鼠标点击后等待程序响应"成为强需求时可能需要加 `--wait` 选项。

3. **drag 使用线性插值而非要求 agent 提供完整路径**：减少 agent 调用复杂度。插值步数默认 5，足以覆盖大多数 TUI 拖拽场景。若某些程序需要精确曲线路径，agent 可用 press/move/release 原子操作手动控制。

4. **坐标校验使用 session 当前 rows/cols**：越界返回错误而非静默裁剪。Agent 应先查询尺寸再操作。此决策基于"silent failure 比 explicit error 更难调试"。

5. **click 的 count > 1 产生多次 press+release 对**：模拟双击/三击。每对之间不插入延迟——PTY 程序通过快速连续事件识别多击。

## Failure Modes & Mitigations

| Failure Mode | Impact | Mitigation |
|---|---|---|
| 程序未启用鼠标模式 | 序列被忽略或显示为乱码 | 文档说明；Response 仍返回 ok（fire-and-forget 语义） |
| 坐标越界 | 程序可能异常 | Daemon 校验 x ∈ [1, cols], y ∈ [1, rows]，越界返回 error |
| Session 已退出 | 写入失败 | 检查 exited 状态，返回 error |
| drag steps=0 | 无 move 事件 | 钳位 steps >= 1 |
| 超大 count（如 10000 次滚轮） | 阻塞写入、淹没程序 | 钳位 count <= 100，超出返回 error |

## Implementation Steps

### Step 1: 新增 `crates/core/src/mouse.rs` 编码模块

创建文件，实现以下公开函数：

- `pub fn encode_press(button: MouseButton, x: u16, y: u16) -> Vec<u8>` — 生成 `\x1b[<{btn};{x};{y}M`
- `pub fn encode_release(button: MouseButton, x: u16, y: u16) -> Vec<u8>` — 生成 `\x1b[<{btn};{x};{y}m`
- `pub fn encode_click(button: MouseButton, x: u16, y: u16, count: u16) -> Vec<Vec<u8>>` — count 次 press+release
- `pub fn encode_scroll(direction: ScrollDirection, x: u16, y: u16, count: u16) -> Vec<Vec<u8>>` — count 次滚轮按下+释放
- `pub fn encode_move(button: MouseButton, x: u16, y: u16) -> Vec<u8>` — 生成 motion 事件（button+32 编码）
- `pub fn encode_drag(button: MouseButton, from_x: u16, from_y: u16, to_x: u16, to_y: u16, steps: u16) -> Vec<Vec<u8>>` — press + interpolated moves + release

定义枚举：
- `pub enum MouseButton { Left, Middle, Right }` — 映射到 SGR button code 0/1/2
- `pub enum ScrollDirection { Up, Down }` — 映射到 64/65

在 `crates/core/src/lib.rs` 中添加 `pub mod mouse;`。

**测试验证**：`cargo test -p agent-shell-core` 通过 7 个新单元测试。

### Step 2: `Session` 新增 `send_raw_bytes` 方法

在 `crates/core/src/session.rs` 中添加：

```rust
pub fn send_raw_bytes(&mut self, data: &[u8]) -> Result<(), String> {
    let mut writer = self.pty_writer.lock().unwrap();
    writer.write_all(data).map_err(|e| format!("write failed: {}", e))?;
    writer.flush().map_err(|e| format!("flush failed: {}", e))?;
    drop(writer);
    if let Some(ref mut rec) = self.recording {
        rec.record_in(data);
    }
    Ok(())
}
```

**测试验证**：现有测试不回归。

### Step 4: Daemon 处理 `Request::Mouse`

在 `crates/daemon/src/server.rs` 的请求分发中添加 `Request::Mouse` 匹配分支：

1. 查找 session（不存在 → error）
2. 检查 `exited`/`destroying`（已退出 → error）
3. 校验坐标：`x ∈ [1, session.cols]`, `y ∈ [1, session.rows]`（越界 → error）
4. 校验 count <= 100（超出 → error）
5. 解析 action + button/direction → 调用 `mouse::encode_*` 函数
6. 遍历返回的 `Vec<Vec<u8>>`，对每段调用 `session.send_raw_bytes(&bytes)`
7. 返回 `Response::ok()`

**测试验证**：daemon 编译通过。
