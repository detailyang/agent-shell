# Design: Recording & Replay Audit

Generated by /think on 2026-05-15
Status: Draft

## Problem

录制（`--record`）和回放（`replay`）功能已实现但未经严格验收。手动审查发现两个缺陷和测试覆盖不足：

1. **录制路径不隔离**：`Config::recording_dir()` 使用硬编码的 `~/.agent-shell/recordings`，不尊重 `AGENT_SHELL_HOME` 环境变量。e2e 测试中录制文件写入全局目录，测试间互相污染，且遗留垃圾文件。
2. **timed_replay 丢失终端转义**：`escape_for_terminal()` 把所有非 ASCII 可打印字节转义为 `\xHH` 格式，导致回放时颜色、光标移动、VT100 序列全部丢失，终端显示为乱码文本而非真实会话还原。
3. **测试覆盖薄弱**：现有测试只验证"录制文件非空"和"replay 不崩溃"，不验证录制内容正确性（事件方向、数据完整性）或回放输出。

## Out of Scope

- 新增 `--annotate` flag（可作为后续独立 feature）
- 录制格式变更（NDJSON + base64 格式保持不变）
- 跨平台录制兼容性

## Constraints

1. 不修改 `RecordingEvent` 结构体（NDJSON 格式不变）
2. 不修改现有 CLI 子命令签名（`replay FILE [--speed F] [--dump] [--force]`）
3. `AGENT_SHELL_HOME` 覆盖行为必须与 `base_dir()` / `socket_path()` 一致
4. 现有 169 个测试不回归

## Assumptions

1. `timed_replay` 的目标用户是在终端中观看会话回放的人类。输出原始字节（让终端自行渲染）是正确行为。
2. `--dump` 模式已正确输出原始字节，无需修改。
3. `timed_replay` 不再显示 `[INPUT]` 标注。如果未来需要输入可视化，通过新增 `--annotate` flag 实现。

## Approach

### Fix 1: `recording_dir()` 尊重 `AGENT_SHELL_HOME`

修改 `Config::recording_dir()` 逻辑：
- 若 `AGENT_SHELL_HOME` 已设置 → 返回 `$AGENT_SHELL_HOME/recordings/`
- 若 `recording.dir` 为默认值 → 返回 `base_dir()/recordings/`
- 若 `recording.dir` 被用户显式配置 → 尊重用户配置（tilde 展开）

实际上最简单的做法是：当 config 中 `recording.dir` 仍为默认值时，使用 `base_dir().join("recordings")`。这样 `AGENT_SHELL_HOME` 自动生效。

### Fix 2: `timed_replay` 输出原始字节

- 删除 `escape_for_terminal()` 函数
- `timed_replay` 中 `"out"` 事件直接 `write_all(&bytes)`（与 `dump_replay` 一致）
- `"in"` 事件静默跳过（不再显示 `[INPUT]`）
- 保留事件间的时间延迟逻辑

### Fix 3: 加强测试覆盖

新增单元测试和集成测试，验证：
- 录制文件包含正确的 in/out 事件和可解码数据
- `AGENT_SHELL_HOME` 下的录制路径隔离
- `replay --dump` 输出包含预期文本
- `replay` 对空文件和损坏文件的容错

## Key Decisions

1. **选择基于 `base_dir()` 而非引入新的 env var**：复用已有的 `AGENT_SHELL_HOME` 机制，保持一个旋钮控制所有路径。只要 `AGENT_SHELL_HOME` 语义不变，此决策有效。

2. **`timed_replay` 默认不显示输入事件**：输入事件在原始回放中会破坏终端渲染（因为显示 `[INPUT] ...` 文本会与 VT100 序列交错）。未来可通过 `--annotate` 恢复。

3. **不为 timed_replay 加 `--force` 保护**：`timed_replay` 带延迟输出，用于终端观看，pipe 到文件场景用 `--dump`。两种模式目标明确，无需额外保护。

## Failure Modes & Mitigations

| Failure Mode | Mitigation |
|---|---|
| 录制文件为空（session 创建后立即 destroy） | 已有行为：产生 0 行 NDJSON 文件，replay 正常处理（无输出） |
| 录制文件损坏（部分 JSON 行） | `serde_json::from_str` 返回 Err → 跳过该行，已有行为 |
| `AGENT_SHELL_HOME` 目录不存在 | `Recording::new()` 已有 `create_dir_all`，无需修改 |
| timed_replay 输出二进制到非终端 | 用户应使用 `--dump --force`。timed_replay 的延迟输出本身不适合 pipe |

## Test Coverage

### 单元测试 (`crates/core/src/config.rs`)
1. `recording_dir_respects_agent_shell_home` — 设置 `AGENT_SHELL_HOME` → `recording_dir()` 返回其下 `recordings/` 子目录
2. `recording_dir_default_without_env` — 无 `AGENT_SHELL_HOME` → 返回 `~/.agent-shell/recordings`

### 集成测试 (`crates/e2e/tests/integration.rs`)
3. `recording_path_under_agent_shell_home` — 创建 session 时返回的 `recording` 路径在 `AGENT_SHELL_HOME` 下
4. `recording_content_correct` — 创建 session → send "echo test" → destroy → 解析录制文件 → 验证包含 `dir=in` 事件含 "echo test" 和 `dir=out` 事件含 "test"
5. `replay_dump_output_correct` — 创建 session → send → destroy → `replay --dump --force` → stdout 包含预期文本
6. `replay_empty_file` — 对空 JSONL 文件执行 replay → 正常退出，无 crash
7. `replay_corrupted_file` — 对包含非 JSON 行的文件执行 replay → 正常退出，跳过坏行

### 未覆盖（已知 gap）
- `timed_replay` 的延迟精度（依赖 `thread::sleep`，无法精确测试）
- `--speed` 参数对延迟的影响（同上）

## Rollback Plan

所有改动为 bug fix + 测试新增：
1. `recording_dir()` 改动可回滚到使用硬编码路径
2. `timed_replay` 改动可回滚到 `escape_for_terminal` 版本
3. 新增测试可直接删除

无数据迁移，无 schema 变更，无风险。

## Open Questions

无。

## Success Criteria

1. `Config::recording_dir()` 在 `AGENT_SHELL_HOME=/tmp/X` 时返回 `/tmp/X/recordings`
2. `create --record` 返回的 `recording` 路径在 `AGENT_SHELL_HOME` 目录下（e2e 验证）
3. `replay FILE` 的 stdout 包含原始 VT100 转义序列（非 `\xHH` 转义文本）
4. 录制文件内容可被解析，包含正确的 in/out 事件和可解码的 base64 数据
5. 全部现有测试 + 新增测试通过，零回归

## Implementation Steps

### Step 1: 修复 `Config::recording_dir()` 尊重 `AGENT_SHELL_HOME`

修改 `crates/core/src/config.rs` 中 `Config::recording_dir()` 方法：若 `AGENT_SHELL_HOME` 环境变量已设置，返回 `$AGENT_SHELL_HOME/recordings/`；否则使用 `base_dir().join("recordings")`。删除对 `self.recording.dir` 默认值的依赖（仅当用户在 config.toml 中显式设置 `recording.dir` 时才使用该值）。

在 `crates/core/src/config.rs` 的 `#[cfg(test)]` 模块中新增 `recording_dir_respects_env` 测试。

**验证**：`cargo test -p agent-shell-core config` 通过。

### Step 2: 修复 `timed_replay` 输出原始字节

修改 `crates/core/src/recording.rs`：
- `timed_replay` 函数中，`"out"` 事件直接 `write_all(&bytes)` + `flush`，不调用 `escape_for_terminal`
- `"in"` 事件不输出任何内容（静默跳过）
- 删除 `escape_for_terminal` 函数（不再有调用者）

**验证**：`cargo build -p agent-shell-core` 编译通过；手动 `replay` 确认终端正确渲染。

### Step 3: 新增集成测试

在 `crates/e2e/tests/integration.rs` 中，替换现有 `mod recording_verify` 和 `mod replay` 测试模块为加强版本，包含 5 个新测试（见 Test Coverage 部分的 #3–#7）。

**验证**：`cargo test -p agent-shell-e2e recording` 和 `cargo test -p agent-shell-e2e replay` 全部通过。

### Step 4: 全量回归测试

运行 `cargo test` 确认所有测试通过，无回归。
